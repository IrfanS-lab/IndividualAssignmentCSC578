What is Git

Git is a mature, actively maintained open source distributed version control system developed in 2005 by Linus Torvalds.

Unlike older centralized systems, every developer's working copy of the code is a full-fledged repository that can track changes independently.

git --version
Initializing a Repository

To begin tracking a local workspace folder with version control, change directories into your project folder and run the init command sequence.

This generates a hidden folder where Git caches all background snapshot tree changes securely.

git init
  • Creates a hidden directory called .git.
  • Sets up a default branch, typically named main.
Staging and Committing

Recording adjustments happens via a two-step handshake: staging files to the index, then saving that bundle down to the timeline history record.

The staging layer lets you fine-tune exactly what modifications roll into your next snapshot release bundle.

git add index.html styles.css

Once you are satisfied with the items in your staging area, seal them permanently with a brief, clear message:

git commit -m "feat: implement initial web canvas structure"
Branching Basics

Branching isolated pathways lets team members safely craft individual code features without fouling the primary operational trunk state.

You can seamlessly isolate experimental structures, delete broken branches, or pull verified merges back into production whenever you choose.

git checkout -b feature-navigation
  • Isolates feature work from production routines.
  • Allows parallel deployment work streams.
  • Simplifies history pruning operations.
Remote Repositories

To back up your modifications and collaborate across teams, you link your machine's workspace database tracker to a remote cloud host profile.

Common upstream host servers include platforms like GitHub, GitLab, and Bitbucket.

git push -u origin main
